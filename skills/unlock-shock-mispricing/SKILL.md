# Unlock Shock Mispricing

## 目标输出

围绕“解禁后短期冲击是否被高估？多久完成价格修复？”输出以下内容：

1. **冲击是否被高估**：高/中/低概率。
2. **修复窗口**：预估 `T+5 / T+10 / T+20 / T+60` 的修复概率与基准区间。
3. **证据链**：四类信号（解禁规模、大宗交易、股东减持、资金流）的同向/背离关系。
4. **交易与风控建议**：入场触发条件、失效条件、仓位节奏。

## 输入清单

至少收集以下字段（缺失要标注）：

- 解禁事件：解禁日期、解禁股数、解禁市值、占流通盘比例、涉及股东类型。
- 大宗交易：解禁日前后 20 个交易日的大宗成交金额、折价率、买卖席位集中度。
- 股东减持：公告减持计划、实际减持进度、减持主体（控股股东/高管/财务投资者）。
- 资金流：主力净流入、超大单净额、北向资金（若适用）、换手率与量价关系。
- 市场背景：指数环境、行业相对强弱（用于剔除系统性影响）。

## 四步工作流

### 第一步：解禁压力定标（Supply Shock）

将解禁冲击先量化为基础压力分：

- 低压力：解禁市值/日均成交额 < 3
- 中压力：3~7
- 高压力：> 7

再看结构：

- 若主要为产业股东、且历史减持意愿低，下调一级压力。
- 若主要为财务投资者、且浮盈高，上调一级压力。

### 第二步：真实抛压确认（Behavior Check）

用大宗交易 + 减持行为确认“预期抛压”是否真的落地：

- **预期高、行为低**（错杀候选）：
  - 大宗成交温和、折价收敛；
  - 减持公告有但执行慢或未显著放量；
  - 二级市场未出现持续放量下跌。
- **预期高、行为高**（非错杀）：
  - 大宗连续高折价；
  - 重要股东快速减持；
  - 放量下跌并跌破关键均线。

### 第三步：资金承接验证（Demand Absorption）

判定是否存在“接盘资金”：

- 主力资金连续 3~5 日净流入为正；
- 下跌缩量、反弹放量；
- 大跌日尾盘回收明显（承接特征）。

若供给冲击高但承接更强，可把“错杀概率”上调。

### 第四步：修复时间估计（Repair Window）

按信号组合估计修复窗口：

- **快修复（5~10个交易日）**：行为抛压弱 + 资金承接强。
- **中修复（10~20个交易日）**：行为抛压中等 + 资金承接一般。
- **慢修复（20~60个交易日）**：行为抛压强或市场风格逆风。

优先给区间而不是单点天数，并附带触发条件与失效条件。

## 打分框架（建议）

按 0-100 分评估“错杀修复分（MSR）”：

- 解禁基础压力（20分，反向计分）
- 大宗交易友好度（25分）
- 减持行为温和度（25分）
- 资金流承接强度（30分）

阈值示例：

- `MSR >= 70`：错杀概率高，优先跟踪 T+5/T+10 修复。
- `50 <= MSR < 70`：中性，等待二次确认。
- `MSR < 50`：谨慎，偏向真实出清压力。

需要快速计算时，优先运行 `scripts/score_unlock_repair.py`。

## 输出模板

使用以下结构输出结论：

1. **一句话结论**（是否错杀 + 修复窗口）
2. **核心证据**（四类信号逐条）
3. **情景推演**（乐观/中性/悲观）
4. **执行计划**（观察位、加减仓触发、止损失效条件）
5. **待补数据**（缺失项与对结论的影响）
