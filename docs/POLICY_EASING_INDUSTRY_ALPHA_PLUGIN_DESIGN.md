# 政策宽松到行业超额收益传导链插件设计文档

## 1. 文档信息

- 状态：Proposal
- 日期：2026-03-26
- 目标插件：`policy_easing_industry_alpha`
- 主题：基于 **LPR/新增贷款/货币供应量 + 行业资金流 + 行业研报 + 估值** 的宽信用行业轮动规律发现

---

## 2. 背景与核心问题

在宽信用周期中，市场通常不是“全行业同步上涨”，而是沿着“政策→信用→流动性→风险偏好→估值重估→盈利兑现”的链条分阶段演绎。

本插件专注回答一个高价值问题：

> **宽信用周期里，哪些行业呈现“先涨估值，后涨盈利”的可重复规律？**

现有研究常见短板：

1. 宏观变量与行业收益的映射只看同频相关，缺少“时滞”与“路径分解”。
2. 只做因子回归，不拆“估值扩张 vs 盈利增长”的收益来源。
3. 只看价格和财务，不纳入资金流与研报预期变化，导致传导链不闭环。
4. 缺少可运营化插件，难以持续更新与复盘。

---

## 3. 设计目标

### 3.1 业务目标

1. 自动识别宽信用阶段，并形成可复用的阶段标签。
2. 对行业收益进行归因拆解：
   - 估值驱动（PE/PB/EV-EBITDA 重估）
   - 盈利驱动（EPS/净利润预期与实际改善）
3. 识别“估值先行、盈利滞后”的行业候选池，并输出置信度评分。
4. 提供可解释证据链（宏观触发→资金流共振→研报预期修正→估值/盈利时序）。

### 3.2 工程目标

1. 插件化：提供标准输入、标准输出和可调用工具函数。
2. 可扩展：后续可扩展至“紧信用”或“政策中性”框架。
3. 可验证：内置回测与稳定性检验（滚动窗口、子样本、参数扰动）。
4. 可观测：输出每一步中间特征和诊断日志，便于解释与调参。

## 4. 范围与非目标

### 4.1 范围

- 市场范围：A 股一级/二级行业（可配置）。
- 频率：以月频为主，支持周频增强。
- 数据维度：
  - 宏观：LPR、社融口径新增贷款、M1/M2/社融增速等。
  - 行业：价格、估值、盈利、资金流。
  - 文本：行业研报标题/摘要/评级/目标价变动。

### 4.2 非目标

1. 不在第一期引入个股级选股交易信号。
2. 不做高频日内策略。
3. 不追求一次性解决全部宏观范式，只聚焦“政策宽松→信用扩张”主线。

## 5. 传导链研究框架

### 5.1 假设链条

定义宽信用传导主链：

1. **政策宽松信号启动**（LPR下行、信贷脉冲回升、货币扩张）
2. **无风险利率与贴现率下行预期形成**
3. **风险偏好抬升 + 增量资金流入特定行业**
4. **行业估值先扩张**（特别是久期长、弹性高行业）
5. **分析师预期上修扩散**（覆盖增长、评级、目标价）
6. **盈利改善兑现**（收入/利润/ROE改善）

### 5.2 可检验命题

- 命题 A：宽信用初期，行业超额收益主要由估值扩张贡献。
- 命题 B：估值扩张领先盈利改善 1~4 个季度，且领先长度行业间可区分。
- 命题 C：存在“资金流与研报共振”时，估值先行信号更稳健。
- 命题 D：当信用脉冲回落且盈利未兑现时，先行行业回撤风险显著上升。

## 6. 数据层设计

### 6.1 数据分层

1. **Raw 层**：原始宏观序列、行业行情、估值、财务、资金流、研报文本。
2. **Feature 层**：标准化与时滞对齐后的特征。
3. **Signal 层**：阶段标签、行业评分、候选池。
4. **Insight 层**：规律摘要、行业对比、可视化指标卡。

### 6.2 关键特征构建

#### 6.2.1 宏观宽松特征（Macro Easing Features）

- `lpr_1y_diff_3m`：1年LPR近3个月变动。
- `new_loan_yoy_z`：新增贷款同比的标准分。
- `credit_impulse`：信用脉冲（新增信贷/GDP 或替代口径变化率）。
- `m1_m2_scissor`：M1-M2 剪刀差。
- `liquidity_composite`：以上变量经 PCA/加权合成的流动性指数。

#### 6.2.2 行业资金流特征（Flow Features）

- `northbound_netflow_industry`：北向资金行业净流入。
- `margin_financing_industry`：融资余额行业增速。
- `etf_flow_proxy`：行业ETF净申购代理指标。
- `flow_acceleration`：资金流增速变化（2阶差分/斜率）。

#### 6.2.3 行业研报特征（Research Features）

- `report_count_mom`：研报数量环比。
- `rating_upgrade_ratio`：评级上调比例。
- `target_price_revision`：目标价上修幅度。
- `earnings_revision_diffusion`：盈利预测上修扩散度。
- `theme_alignment_score`：与“宽信用受益”主题的语义匹配分。

#### 6.2.4 估值与盈利拆解特征（Valuation/Earnings Features）

- `pe_zscore`, `pb_zscore`：相对历史分位或标准分。
- `valuation_re_rate_3m`：估值重估速度。
- `eps_fy1_revision_3m`：未来一年 EPS 预期修正。
- `profit_growth_realized`：盈利实际增速。
- `roe_trend`：ROE 趋势变化。

## 7. 方法论设计

### 7.1 宽信用阶段识别（Regime Detection）

采用“规则+统计”双引擎：

1. 规则层：
   - LPR 连续下行或维持低位；
   - 新增贷款同比/信用脉冲进入上行区间；
   - 货币增速条件满足阈值。
2. 统计层：
   - 使用 Markov Switching / HMM 对“宽信用状态概率”建模。

输出：

- `regime_state ∈ {tight, neutral, easing}`
- `easing_probability ∈ [0,1]`

### 7.2 行业收益来源分解

对行业未来超额收益 `R_excess` 做分解：

- `R_excess ≈ ΔValuation + ΔEarnings + ΔRiskPremiumResidual`

其中：

- `ΔValuation` 由估值倍数变化贡献近似。
- `ΔEarnings` 由EPS或净利润预期/实际变化贡献。

通过滚动窗口估计每个行业在不同阶段的“估值贡献占比”。

### 7.3 “先估值后盈利”识别模型

构建两类信号：

1. **先行估值信号**：
   - 宽信用概率高
   - 资金流加速
   - 研报预期扩散向上
   - 估值分位不极端（避免纯泡沫尾段）
2. **滞后盈利确认信号**：
   - 未来 1~4 季 EPS 预测持续上修
   - 盈利实际数据兑现

定义行业模式标签：

- `pattern_A`: 估值先行、盈利后至（目标模式）
- `pattern_B`: 估值与盈利同步
- `pattern_C`: 仅估值扩张，盈利未兑现
- `pattern_D`: 盈利改善但估值未反应

### 7.4 时滞与因果校验

- Cross-correlation：估计宏观→资金→估值→盈利的领先滞后期。
- Granger causality（谨慎解释）：验证增量预测关系。
- Local projection：估计宽信用冲击下行业多期响应路径。

## 9. 规律发现机制（重点）

为满足“深入可以发现规律”，插件不只输出排名，还输出三类规律：

1. **时序规律**：
   - 宏观变量变化后，哪些行业在第 `t+1/t+2/...` 期先出现估值扩张。
2. **结构规律**：
   - 哪些行业在多个宽信用子周期中重复出现 pattern_A。
3. **失效规律**：
   - 哪些条件下 pattern_A 会失效（例如估值过高且资金流转负）。

### 9.1 规律挖掘技术方案

- 周期切片：按历史宽信用窗口切分样本。
- 事件研究：以政策宽松事件为锚点做行业累计超额收益曲线。
- 子群稳健性：按估值高低、行业属性（成长/周期/金融）分层。
- 规则提炼：使用可解释模型（GAM/浅层树）输出人类可读规则。

示例规则（自动生成）：

- 当 `easing_probability > 0.65` 且 `flow_resonance > 0.7` 且 `rating_upgrade_ratio > 60%` 时，
  行业未来两个季度出现 `pattern_A` 的概率提升至基线的 1.8 倍。

## 10. 验证与评估

### 10.1 离线评估

1. 识别准确性：
   - Pattern 分类准确率、F1、AUC。
2. 投资有效性：
   - 行业组合超额收益、信息比率、最大回撤。
3. 解释一致性：
   - 规则跨窗口重复出现比例。

### 10.2 稳健性测试

- 滚动起点、窗口长度变化。
- 指标口径替代（例如社融替代新增贷款）。
- 去除单一数据源后性能衰减测试。

### 10.3 反事实检查

- 将“研报特征”置零，观察模式识别能力下降幅度。
- 将“资金流特征”置零，检验估值先行信号是否退化。

## 11. 风险与治理

1. **数据修订风险**：宏观与盈利数据存在回溯修订，需要版本化存档。
2. **主题漂移风险**：研报语义词典需要季度更新。
3. **过拟合风险**：限制模型复杂度，保留朴素基线。
4. **解释风险**：Granger 仅代表预测先后，不直接等同因果。

治理措施：

- 数据快照与可追溯日志。
- 规则库版本控制。
- 每月自动再训练 + 漂移监控告警。
