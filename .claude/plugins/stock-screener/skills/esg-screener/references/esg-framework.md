# 治理与风险代理框架

目标是在当前仓库能力内，用可验证的治理、合规和资本配置代理信号替代伪完整 ESG 评分。

## 1. 核心原则

- 不把当前仓库描述成完整 ESG 数据平台
- 以 `G` 为主轴，`E` 和 `S` 通过争议、处罚、披露与外部评级做代理
- 先识别风险，再识别改善机会
- 对 `soe-rerating`，优先验证治理改善是否真实，而不是只看题材叙事

## 2. 代理指标框架

### 2.1 治理（G）优先

优先看：
- 股东结构是否稳定
- 分红、回购与资本配置是否一致
- 审计意见是否异常
- 是否频繁收到监管措施或问询函
- 是否存在中小股东保护与治理改善线索
- 国企的市值管理、资产整合、激励约束是否有公开线索

### 2.2 环境与社会（E/S）以风险事件为代理

在当前仓库内，更可信的做法是通过以下信号做代理：
- 行政处罚、问询、监管措施
- 重大争议或披露缺口
- 行业本身的环境敏感度
- 外部 ESG 评级变动

### 2.3 `soe-rerating` 治理验证顺序

默认按以下顺序验证：
1. 先确认 `high-dividend-strategy` 已识别分红增强、派息率提升或市值管理预期
2. 再核对股东结构是否更稳定、监管 / 问询 / 审计风险是否缓和
3. 再检查是否存在资产整合、激励约束、公开治理优化等可引用线索
4. 最后判断应归入 `国企治理重估`、`治理改善机会`，还是转入 `监管风险预警`

如果只有题材叙事、没有公开资料支撑，不得写成“国企重估已成立”。

## 3. 分类框架

### `治理改善机会`

- 过去治理一般，但近期股东结构、分红政策、监管状态出现改善
- 市场尚未充分反映这种改善

### `国企治理重估`

- 市值管理、分红提升、资产整合、激励约束等治理线索开始明确
- 市场仍按旧估值框架定价

### `稳健治理候选`

- 股东结构稳定
- 分红与资本配置克制
- 监管事件较少
- 治理风险整体可控

### `监管风险预警`

- 监管措施、问询、审计意见或争议事件明显偏多
- 不一定立刻剔除，但必须单独标红

### `高收益但高治理风险`

- 表面估值或股息率有吸引力
- 但股东结构、监管事件或资本配置存在明显隐患

## 4. 异常发现层

每次执行必须写出异常，例如：
- 高股息、高收益率，但问询和监管措施明显偏多
- 治理改善已经出现，但市场仍按旧形象定价
- 国企分红、市值管理或治理改善开始出现，但市场还未充分计价
- 外部评级一般，但股东结构和资本配置比历史更稳
- 行业环境敏感度高，但公司披露和治理跟不上

## 5. 外部评级使用原则

外部 ESG 评级只做补充：
- 用来对照治理代理结论
- 用来观察方向性变化
- 不直接替代内部研究判断

如果外部评级与治理代理相矛盾，优先把矛盾写出来，而不是强行统一成一个结论。

## 6. 跟踪信号模板

对重点候选至少跟踪：
- 监管措施和问询函是否减少
- 审计意见、分红政策、回购和资本配置是否改善
- 大股东和流通股东结构是否稳定
- 市值管理、资产整合、激励约束是否有新进展
- 外部 ESG 评级是否变化
- 是否有新的环境、安全或合规争议暴露
