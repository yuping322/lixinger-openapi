# esg-screener 设计决策

## 1. 核心定位

把 ESG 策略收缩为“治理与风险代理筛选”，避免伪装成仓库当前无法支撑的完整 ESG 评分系统。

它同时承担 `soe-rerating` 的治理验证层：
- 在 `high-dividend-strategy` 负责分红增强和股东回报的同时
- 由本策略负责治理、股东结构、监管风险与资本配置证据

## 2. 候选池底座

统一先用 `.claude/skills/lixinger-screener` 做候选池与基础排除，原因是：
- 先控制范围
- 先排除明显财务与交易性不合格样本
- 把治理和争议补查留给少量入围股

## 3. 代理框架

默认优先看：
- 股东结构
- 分红、回购与资本配置稳定性
- 审计意见、监管措施、问询函
- 国企治理改善、市值管理、资产整合、激励约束线索
- 外部 ESG 评级变化（仅作参考）

这比编造一个无法核验的 E/S/G 综合分更可信。

## 4. 接口选择

优先使用：
- `cn/company/majority-shareholders`
- `cn/company/nolimit-shareholders`
- `cn/company/measures`
- `cn/company/inquiry`
- `cn/company/fs/non_financial` 与 `fundamental/non_financial` 做财务与分红侧辅助

## 5. 输出策略

结果至少拆成：
- `治理改善机会`
- `国企治理重估`
- `稳健治理候选`
- `监管风险预警`
- `高收益但高治理风险`

并且必须说明哪些结论来自外部评级，哪些来自仓库内代理数据。

## 6. 数据边界

当前不做的事情：
- 伪造完整 ESG 评分
- 把未验证的环境、社会指标写成已实装能力
- 在缺少公告或公开资料时断言国企改革一定落地
- 忽略外部依赖和数据缺口

## 7. 第一轮改造清单

按当前优先方向并入设计，后续优先回填以下位置：

1. `SKILL.md`
   - 固定 `soe-rerating` 的治理验证分支，只在股东回报改善线索出现后启用
   - 对高息但高治理风险样本，强制转入风险分类而不是给正向推荐
2. `references/esg-framework.md`
   - 明确股东结构、监管措施、问询函、审计意见、资产整合、激励约束的验证顺序
3. `references/output-template.md`
   - 固定 `治理验证结论`、`公开资料来源`、`监管 / 审计风险`、`待验证改革线索`
4. 与 `high-dividend-strategy` 的协同
   - 先由红利侧判断分红重估，再由本策略确认治理是否支持重估成立
