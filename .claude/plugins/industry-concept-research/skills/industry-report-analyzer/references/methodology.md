# 方法论：行业研报分析（共识-分歧-验证）

## 1. 目标

把多份行业研报转化为可执行的研究框架：
- 找出市场共识
- 识别核心分歧
- 设计验证路径

## 2. 分析框架

### 2.1 共识层

聚焦以下是否在多数研报中重复出现：
- 行业所处周期位置（复苏/扩张/放缓/出清）
- 驱动主线（需求、供给、价格、政策、技术）
- 盈利方向（收入、毛利率、净利率）
- 估值锚（PE/PB/PS/EV-EBITDA）

### 2.2 分歧层

围绕“关键假设”建立分歧矩阵：
- 需求弹性（销量、ASP、渗透率）
- 供给节奏（扩产、出清、产能利用率）
- 政策与监管（补贴、准入、医保、出口）
- 海外变量（汇率、关税、竞争格局）

### 2.3 验证层

每个观点必须绑定可观测变量：
- 数据指标：价格、库存、订单、开工率、招标量
- 时间窗口：下周/月度/季度验证点
- 失效条件：关键指标跌破/不达预期

## 3. 评分建议

可用 1-5 分量化“观点可执行性”：
- 证据强度
- 可跟踪性
- 时效性
- 风险暴露

## 4. 输出原则

- 先结论，后证据。
- 每条结论至少对应一个验证指标和一个失效条件。
- 明确“已验证事实”与“尚待验证假设”的边界。
