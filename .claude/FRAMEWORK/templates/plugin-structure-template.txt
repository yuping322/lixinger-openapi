# 目录结构说明

## 必填目录和文件
1. **README.md**：插件说明文档
2. **plugin.yaml**：插件元信息配置
3. **commands/**：至少包含一个命令文件
4. **skills/**：至少包含一个 skill 目录
5. **contracts/**：包含 input/output/qa-rules 三个契约文件
6. **tests/smoke/**：至少包含一个冒烟测试文件

## 可选目录和文件
1. **skills/<skill>/references/**：参考资料
2. **skills/<skill>/scripts/**：执行脚本
3. **skills/<skill>/examples/**：示例文件
4. **templates/**：输出模板
5. **tests/integration/**：集成测试
6. **tests/performance/**：性能测试
7. **docs/**：详细文档

## 文件命名规范
- 目录名：小写字母，单词间用连字符 `-` 连接（如：`risk-monitor`）
- 文件名：小写字母，单词间用下划线 `_` 连接（如：`input.schema.json`）
- Skill 目录名：与 skill 名称一致（如：`valuation-detector`）
- Command 文件名：与命令名称一致（如：`query-valuation.md`）

## 契约文件规范
所有契约文件必须使用 JSON Schema 格式，包含：
- `$schema`：JSON Schema 版本
- `title`：契约标题
- `description`：契约描述
- `type`：数据类型
- `properties`：字段定义
- `required`：必填字段列表
- `examples`：示例数据

## 测试文件规范
每个测试文件必须包含：
- **测试场景**：描述测试目的
- **输入参数**：测试输入
- **预期输出**：期望结果
- **预期置信度**：期望的置信度范围
- **测试结果**：实际结果（通过/失败）
