# 输出模板：组合监控流程编排器

> 目标：将组合暴露、风险、流动性、事件与再平衡需求整合成统一监控报告。

## 组合监控总报告

### 1) 结论摘要（3–5条）

- 当前组合整体状态为 `[健康 / 偏进攻 / 偏防守 / 风险偏高]`。
- 组合主要暴露集中在 `[行业 / 风格 / 单一主题 / 个股集中度 / 宏观因子]`。
- 当前最大风险来自 `[集中度 / 相关性抬升 / 流动性不足 / 高波动暴露 / 事件冲击]`。
- 与目标基准或策略约束相比，当前偏离主要体现在 `[权重 / 风格 / 回撤 / 波动]`。
- 后续优先动作应为 `[观察 / 减仓 / 再平衡 / 对冲 / 补充研究]`。

### 3) 分析与解释

#### 3.1 暴露与集中度

- 组合是否过于集中在少数行业、主题或单一个股。
- 风格偏好是主动选择还是被动漂移。
- 若市场风格突变，哪些持仓最敏感。

#### 3.2 风险状态

- 持仓之间相关性是否上升，导致分散化失效。
- 高波动、低流动性、事件驱动型持仓的占比如何。
- 是否存在看似分散、实则同一宏观因子暴露的问题。

#### 3.3 监控动作

- 哪些持仓需要重点跟踪公告、财报或资金变化。
- 哪些仓位需要设定止损、减仓或观察阈值。
- 是否需要启动再平衡或对冲流程。

### 4) 风险清单与监控指标

| 风险 | 触发条件 | 监控指标 | 应对建议 |
|---|---|---|---|
| 持仓过度集中 | 前五大权重过高 | 集中度、单股权重 | 分散暴露 |
| 相关性抬升 | 分散化效果下降 | 相关性矩阵 | 增加非相关资产 |
| 流动性风险 | 中小盘或低成交持仓偏多 | 日均成交额、冲击成本 | 下调执行规模 |
| 事件风险 | 财报、减持、监管集中出现 | 事件日历 | 提前准备预案 |

### 5) 下一步（需要补充/核验）

- 建立按日/周/月更新的组合监控面板。
- 对高风险仓位联动个股体检、流动性、股东与公告分析。
- 如有基准约束，补做偏离归因与回撤来源拆解。

### 6) 缺失数据与降级说明

- 缺失数据：实时持仓变动、盘中风险暴露、组合内对冲头寸细节。
- 替代口径：使用最新持仓、历史收益序列与公开事件信息。
- 对结论置信度影响：中。
