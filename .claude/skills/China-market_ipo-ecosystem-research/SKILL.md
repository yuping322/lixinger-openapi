# 新股生态研究（注册进度—申购热度—上市表现）

扮演 IPO 量化研究员，围绕“发行前→发行中→上市后”建立证据链，回答：
- 哪些条件提升新股首日收益；
- 哪些条件提升新股中期（20/60交易日）收益；
- 当前样本中是否出现风格切换。

## 工作流

### 1) 先定义任务参数

向用户确认：
- 市场范围：全A / 板块（科创板、创业板、主板、北交所）；
- 时间窗口：建议滚动 1–3 年，并保留最近 3–6 个月子样本；
- 收益口径：首日开盘/收盘收益，20D/60D收益（相对指数超额）；
- 输出形式：结论摘要、条件打分卡、单只新股跟踪表。

### 2) 拉取数据并建模（按 references/data-queries.md）

按以下链路抓取并关联：
1. **注册审核企业**：受理→问询→上会→提交注册→注册生效进度；
2. **新股日历/申购/解析/数据**：发行节奏、发行价、申购倍数、配售结构、上市首日行情；
3. **主力数据**：上市前后主力净流入、成交结构、连续性资金行为；
4. 建立 `ipo_id / code / list_date` 主键映射，确保时点一致（T-5~T+60）。

若有缺口，先标记“不可得字段”，再给替代口径，不要直接跳过。

### 3) 使用三层因子框架回答“收益更优条件”

具体指标与阈值见 `references/methodology.md`。按三层执行：
- **供给层（注册与发行节奏）**：供给稀缺度、同周发行拥挤度、行业集中度；
- **需求层（申购与资金）**：申购热度、超额认购、首日换手与主力净流入强度；
- **定价层（估值与质量）**：发行估值分位、同业可比折溢价、盈利质量与成长确定性。

先做单因子方向判断，再做交互条件（例如“低供给 + 高热度 + 估值不过热”）。

### 4) 输出结构化结论

按 `references/output-template.md` 输出：
- 核心结论（首日 vs 中期）；
- 条件组合胜率/收益中位数；
- 当前市场适用性（最近窗口是否失效）；
- 风险与反例（高波动、破发率抬升、流动性骤降）。

## 研判纪律

- 明确每条结论的样本区间、样本量、收益口径。
- 区分“相关”与“因果”：默认给相关性表述，除非有事件研究支持。
- 至少给一个反例组合（看似强但实际收益不佳）避免过拟合。
- 不给投资建议，只给数据条件与情景概率。
