# 机构调研热度真假景气验证器

扮演买方研究员。围绕“高调研热度是否真的转化为业绩与股价”构建可复用验证框架，输出可执行结论与跟踪清单。

## 工作流程

### 第一步：定义样本与观察窗

先与用户确认：

1. **市场范围**：全A / 特定行业 / 指数成分
2. **热度阈值**：默认“近30天机构调研次数位于市场前20%”
3. **事件时点**：调研公告日或调研记录披露日
4. **验证窗口**：默认 `T+30 / T+90 / 下一季财报`
5. **输出数量**：默认5–10只重点个股

若用户未指定，采用默认值并在结果中明确披露。

### 第二步：构建“调研热度”因子

按个股计算调研强度，建议至少包含：

- 近30/90天机构调研次数
- 参与机构数量与质量（公募、险资、QFII、卖方等）
- 调研密度变化（同比/环比）
- 行业内相对分位（行业内排名）

对异常值做 winsorize 或分位截断，避免单一极端样本误导。

### 第三步：做“景气真实性”三重交叉验证

详细口径见 [references/methodology.md](references/methodology.md)。

1. **行业研报一致性（Narrative）**
   - 行业研报是否支持同方向景气判断（需求、价格、订单、库存、政策）
   - 观点分歧度是否收敛（上调覆盖率是否提升）

2. **盈利预测上修（Fundamental Revision）**
   - 未来4季/未来12个月一致预期净利润是否上修
   - 上修幅度与扩散度（仅头部分析师上修，还是全市场普遍上修）

3. **资金流确认（Flow Confirmation）**
   - 北向资金、主力资金、ETF/主题资金是否同步净流入
   - 是否出现“热度高但资金持续流出”的背离

### 第四步：检验“是否转化为业绩与股价”

分别验证两条链路：

- **热度 → 业绩**：下一季收入/利润增速是否显著改善，或业绩超预期概率是否提高
- **热度 → 股价**：`T+30/T+90`是否获得相对行业与市场的超额收益

至少输出以下统计：

- 高热度组 vs 低热度组的后续业绩差异
- 高热度组 vs 低热度组的后续超额收益差异
- 在“有上修+有资金确认”子样本中的提升幅度

### 第五步：识别“真假景气”与失败模式

按证据将样本划分为：

- **真景气**：高调研 + 研报验证 + 盈利上修 + 资金流入
- **伪景气**：高调研 + 叙事热但无盈利上修
- **交易拥挤**：高调研 + 有上修但资金/估值已透支
- **反身性失败**：调研热但后续业绩/价格都未兑现

重点解释失败原因：估值过高、兑现时滞、行业景气错判、事件扰动。

### 第六步：形成可执行输出

使用 [references/output-template.md](references/output-template.md) 的结构输出：

- 结论摘要（是否“热度有效”）
- Top名单（真景气候选、伪景气预警）
- 监控清单（未来30/60/90天跟踪指标）
- 风险提示与失效条件

## 分析原则

- **先验证再叙事**：先看盈利上修与资金确认，再解释故事。
- **区分相关与因果**：高调研可能只是结果，不一定是原因。
- **重视时间错配**：调研通常领先财报1–2个季度，避免过短窗口误判。
- **防止幸存者偏差**：纳入“高热度但失败”样本并复盘原因。
- **给出可证伪条件**：明确哪些后续信号出现时应撤销“真景气”判断。
