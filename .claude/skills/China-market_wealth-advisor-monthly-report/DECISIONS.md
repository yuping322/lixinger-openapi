# 架构决策记录

## 创建日期：2026-04-04

## 第一阶段：Skill 设计

### 定位：月度综合资产配置分析报告

**设计理由**：
- 用户原始 prompt 是一个完整的财富顾问角色 + 分析框架 + 报告模板
- 将其拆分为 SKILL.md（入口）、methodology.md（方法论与指标体系）、output-template.md（输出模板）、data-queries.md（数据获取）
- 符合现有 skill 架构的模块化设计

**与现有 skill 的关系**：
- `weekly-market-brief-generator`：周度 A 股简报，聚焦 A 股市场
- `wealth-advisor-monthly-report`：月度综合报告，覆盖宏观 + A股 + 债券 + 黄金外汇 + 配置建议
- 两者互补，不重复

### 核心原则提取

从用户 prompt 中提取了四大核心工作原则：
1. 信息确定性原则 — 只用已发生数据
2. 数据可验证原则 — 必须注明日期与来源
3. 分析实用性原则 — 穿透数据提炼逻辑
4. AI 增强分析方法 — 指标交叉验证

**权衡**：
- 严格禁止预测性措辞是本 skill 的核心差异化特征
- 这要求数据获取必须精确到具体日期，增加了数据依赖复杂度

## 第二阶段：数据源设计

### 数据分层

**决策**：将数据源分为两层
1. **理杏仁 API**：A 股指数行情、估值数据（可自动获取）
2. **WebSearch/WebFetch**：宏观经济、债券、黄金外汇数据（需在线搜索）

**理由**：
- 理杏仁 API 覆盖 A 股数据最完整
- 宏观、债券、黄金等数据需多渠道获取
- 分层设计使数据缺失时可降级处理

## 第三阶段：输出模板设计

### 六大模块结构

保留了用户原始 prompt 的六模块结构：
1. 宏观复盘
2. A 股复盘
3. 债券复盘
4. 黄金外汇复盘
5. 综合配置建议
6. 核心风险与监测

**增加**：
- 第 7 模块：缺失数据与降级说明
- 第 8 模块：免责声明
